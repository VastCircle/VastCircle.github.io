clangd

以 gem5 为例子 ，

通过bear scons build/RISCV/gem5.debug -j128 生成 complie_command.json

在json中加入以下配置 ,然后重启就可以使用了

  "clangd.syncFileEvents": true,
  "C_Cpp.intelliSenseEngine": "disabled",
  "C_Cpp.autocomplete": "disabled",
  "clangd.path": "/usr/bin/clangd-18",
  "files.maxMemoryForLargeFilesMB": 4096, // 增加内存对打文件的支持

    "clangd.arguments": [
        "--background-index",
        "--compile-commands-dir=${workspaceFolder}",  //compile_command.json相对路径，cmake默认生成在build，自行配置
        "-j=128",
        "--all-scopes-completion",
        "--completion-style=detailed",
        "--header-insertion=iwyu",
        "--pch-storage=memory",
        "--cross-file-rename",
        "--enable-config",
        "--fallback-style=WebKit",
        "--pretty",
        "--clang-tidy"
        // 网上别人配置clang++，但我这边windows、linux实测不加这行也没啥问题，可能mac可能需要另外加
    ],
