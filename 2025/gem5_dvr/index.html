向量化

通过字段mop 支持3种访存模式 ,通过lumop 进行进一步划分访存详情

unit-stride

Whole register load/store

顾名思义就是以全寄存器进行load/store访存，而不再是element。load可以配置width来确定EEW，而store则固定是8bits。指令显式指定NFIELDS和EEW，此时的LMUL和SEW无效，NFIELDS只支持1、2、4、8（保持是2的幂次），所以操作的真实有效的element总量为：evl=NFIELDS*VLEN/EEW。

比方说vl4re8.v , NF = 4 , EEW = 8 , 所以就是 读取 4 * vlen/ 8 个字节

mask load/store

连续访问ceil(vl/8)的byte，直接向memory传送mask。（其实就是vl长度的bits，因为每个bit对应一个element）

自定义指令学习

gem5 的指令要看/arch/riscv/isa那里面的文件

解码块以关键字decode为开始，后接要解码的指令位域。解码块以关键字decode为开始，后接要解码的指令位域。指令位域在

decode OPCODE {
  0: add({{ Rc = Ra + Rb; }});
  1: sub({{ Rc = Ra - Rb; }});
}

指令format

指定指令格式有两种方式。

一种是在助记符前给出显示的格式规范，用双冒号(::)分隔。比方说 ArmMultAndMultAcc::armMultAndMultAcc();

ArmMultAndMultAcc为格式，armMultAndMultAcc()为助记符，然后就会去找armMultAndMultAcc()

解析器会找到format ArmMultAndMultAcc定义的位置， 然后就会转化为c++

decode COND_CODE {
0xF: ArmUnconditional::armUnconditional();
default: decode ENCODING {
format DataOp {
    0x0: decode SEVEN_AND_FOUR {
        1: decode MISC_OPCODE {
            0x9: decode PREPOST {
                0: ArmMultAndMultAcc::armMultAndMultAcc();
                1: ArmSyncMem::armSyncMem();
            }
            0xb, 0xd, 0xf: AddrMode3::addrMode3();
        }

另一种指定指令格式的方式如format DataOp所示，只需声明一次，解析到每个指令时，都会调用该format所定义的python语句。

format定义

format本质是一段python函数，接受指令定义提供的参数(在解码块中找到)，并生成最多4段c++代码。c++代码块的区别在于它们在生成的输出中出现的位置。

“header output”：生成的代码位于decoder.hh，所有生成的源文件，如decoder.cc和.cc文件里都会include它。header output通常会包含C++和指令相关的类的声明。 ‘‘decoder output’’ ：decoder.cc。包括构造函数、方法定义等（如反汇编）。 ‘‘exec output‘’：指令类的执行方法函数execute() ‘‘decode block’‘：解码块包含一个语句块，将内嵌在解码函数中。一旦解码块指定的位域的值对应上，就会执行这些语句，并返回适当的指令对象。

Template 定义

源码位于src\arch\arm\isa\formats\basic.isa

def format BasicOp(code, *flags) {{
        iop = InstObjParams(name, Name, 'ArmStaticInst', code, flags)
        header_output = BasicDeclare.subst(iop)
        decoder_output = BasicConstructor.subst(iop)
        decode_block = BasicDecode.subst(iop)
        exec_output = BasicExecute.subst(iop)
}};

源码位于src\arch\arm\isa\templates\basic.isa
// Basic instruction class declaration template.
def template BasicDeclare {{
        /**

   * Static instruction class for "%(mnemonic)s".

          class %(class_name)s : public %(base_class)s
            {
             public:
     /// Constructor.
     %(class_name)s(ExtMachInst machInst);
     Fault execute(ExecContext *,
                   Trace::InstRecord *) const override;
           };
     }};
     format中InstObjParams是isa_parser.py中的类，后面也会提及。

模板%(mnemonic)、%(class_name)s都会由InstObjParams构造的iop中的参数进行替换。

BasicConstructor、BasicDecode、BasicExecute与之类似，都是定义了一套含有可变参数的模板。

模板在解析器中表示为Python对象。模板通常通过调用模板对象的subst()方法来生成字符串。该方法接受一个参数，指定模板中替换符号(例如，%(class_name)s)到特定值的映射。

def format CROp(code, *opt_flags) {{
    iop = InstObjParams(name, Name, 'CompRegOp', code, opt_flags)
    header_output = BasicDeclare.subst(iop)
    decoder_output = BasicConstructor.subst(iop)
    decode_block = BasicDecode.subst(iop)
    exec_output = BasicExecute.subst(iop)
}};

// Basic instruction class declaration template.
def template BasicDeclare {{
    //
    // Static instruction class for "%(mnemonic)s".
    //
    class %(class_name)s : public %(base_class)s
    {
      private:
        %(reg_idx_arr_decl)s;

      public:
        /// Constructor.
        %(class_name)s(ExtMachInst machInst);
        Fault execute(ExecContext *, trace::InstRecord *) const override;
        using %(base_class)s::generateDisassembly;
    };
}};

// Basic instruction class constructor template.
def template BasicConstructor {{
    %(class_name)s::%(class_name)s(ExtMachInst machInst)
        : %(base_class)s("%(mnemonic)s", machInst, %(op_class)s)
    {
        %(set_reg_idx_arr)s;
        %(constructor)s;
    }
}};


// Basic instruction class execute method template.
def template BasicExecute {{
    Fault
    %(class_name)s::execute(ExecContext *xc,
        trace::InstRecord *traceData) const
    {
        %(op_decl)s;
        %(op_rd)s;
        %(code)s;
        %(op_wb)s;
        return NoFault;
    }
}};

// Basic decode template.
def template BasicDecode {{
    return new %(class_name)s(machInst);
}};
